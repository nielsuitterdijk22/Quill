# What is Quill

Quill is a self-serve code collaboration platform built for platform teams that need more control than GitHub gives them — without building something from scratch.

You get repositories, pull requests, CI/CD pipelines, and a policy engine that actually lets you enforce how code moves through your system. Branch rules, time windows, change freezes, external approval gates — all composable, all auditable.

Under the hood Quill runs on [Forgejo](https://forgejo.org), a community-maintained Git service. That means your code lives on infrastructure you control, with no vendor lock-in on the git layer.

## Who Quill is for

**Developers** use Quill the same way they'd use GitHub — push code, open PRs, run pipelines. If you know Git and GitHub Actions syntax, you already know most of it.

**Platform and ops teams** use Quill to define how code flows across projects. You set the policies; Quill enforces them.

**Compliance and risk managers** use Quill's policy hierarchy and audit log to answer the hard questions: who merged what, when, and did it go through the right gates.
