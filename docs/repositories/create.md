# Create a repository

## Personal repo

Click **New repository** from the dashboard or the **+** menu. The repo will live under your personal namespace at `quill.so/your-username/repo-name`.

## Org repo

Navigate to your org, then click **New repository**. The repo will live at `quill.so/your-org/repo-name` and will need to be assigned to a project.

## Options

**Name** — the URL slug. Use lowercase letters, numbers, hyphens, and underscores. Must be unique within your namespace.

**Description** — optional. Shows up in repo listings and search results.

**Visibility** — choose who can see the repo:

- *Private* — only you (or org members with access) can see it
- *Internal* — all members of the org can see and clone it
- *Public* — anyone can see it, no account required

**Initialize repository** — creates an initial commit with a `README.md`. Check this if you're starting fresh. Leave it unchecked if you're pushing an existing repo.

**Default branch** — the branch that PRs target by default. `main` unless you change it.

## After creating

You'll land on the empty repo page with copy-paste commands for your first push, or on the file browser if you initialized it.

Org repos are automatically assigned to a project if you created them from within a project context. If you created from the org root, go to repo **Settings → Project** to assign it.
